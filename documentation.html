SaaSify — Documentation

Concise, beginner-friendly instructions to edit and maintain this static Tailwind + HTML landing template.

Introduction

This project is a static SaaS landing page built with HTML, Tailwind CSS (precompiled to css/output.css), and vanilla JavaScript. Files are ready-to-serve from any static host.

Folder Structure Explanation

  • index.html — main page markup.
  • css/output.css — compiled Tailwind stylesheet (do not edit directly if you use Tailwind source files).
  • js/script.js — custom JavaScript (theme toggle, mobile menu, small interactions).
  • assets/images/ — images used by the template (keep names URL-friendly; avoid spaces).
  • tailwind.config.js — Tailwind configuration (optional, if using the source build).
  • package.json — build/dev scripts (if present).

How to Edit Text Content

  1. Open index.html in a code editor.
  2. Find the section you want to update (search for the heading text) and change the HTML content directly.
  3. Keep accessibility attributes: preserve alt on images and semantic tags like header, section, nav, and footer.
  4. For repeated strings (e.g., product name), rename consistently to avoid typos.

How to Change Colors (Tailwind CSS)

There are two common workflows:

  • If you only use classes: Replace Tailwind utility classes in the markup. Example: change bg-blue-600 to bg-pink-600.
  • If you use a Tailwind build: Update tailwind.config.js theme colors, then rebuild the CSS. Typical commands (if project uses npm):
npm install
npm run build

If you don't have a build setup, it's safest to change utility classes in index.html directly and avoid editing output.css manually.

How to Replace Images

  1. Place optimized images inside assets/images/. Prefer WebP or optimized PNG/JPEG.
  2. Avoid spaces or non-URL-safe characters in filenames. Use kebab-case (e.g., logo-weather.webp).
  3. Update the src in index.html to point to the new filename. Example: 
    <img src="assets/images/logo-weather.webp" alt="Company Logo" class="h-6">
  4. Keep or update the alt attribute — meaningful & concise for accessibility.

How to Customize Sections

Sections are defined using semantic elements and utility classes. To change order or hide a section:

  • Move the section's HTML block in index.html to reorder.
  • To hide on small screens, add standard Tailwind classes (for example: hidden md:block).
  • Avoid removing unique IDs (like #pricing, #features) which are referenced by anchor links and scripts.

JavaScript Functionality Overview

Key behaviors and where they live:

  • Theme toggle — toggles .dark on the <html> element and stores preference in localStorage. File: js/script.js.
  • Mobile menu — shows/hides the side menu by toggling classes on the element with id="mobile-menu".
  • Carousel and other small inline scripts — located in index.html near the bottom. They create dots, rotate testimonials, and implement drag/swipe. Avoid renaming referenced classes or IDs.
  • FAQ toggles — simple show/hide toggles using max-height.

If you edit JavaScript, keep safe null checks (verify elements exist with document.getElementById) and guard third-party calls (the template checks for libraries like anime.js).

Deployment / Hosting Instructions

This is a static site. Recommended quick options:

  • Local test: 
    cd path/to/project
python -m http.server 8000
    Then open http://localhost:8000.
  • GitHub Pages: push the repository and enable Pages in settings (use the root branch).
  • Netlify / Vercel: connect the repo; set build step only if you use Tailwind source files. For precompiled output.css no build is necessary.

Ensure the host serves correct MIME types for .webp images and that files retain the same relative paths.

Troubleshooting & Best Practices

  • Always keep backups before bulk changes.
  • Use URL-safe filenames for assets (no spaces).
  • When changing Tailwind config, rebuild CSS and verify no utilities were purged mistakenly.
  • Keep semantic markup and alt text for accessibility checks during review.
